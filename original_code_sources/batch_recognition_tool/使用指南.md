# 批量预识别工具 - 使用指南

## 快速开始

### 1. 安装依赖

```bash
cd batch_recognition_tool
pip install -r requirements.txt
```

### 2. 运行程序

```bash
python main.py
```

### 3. 使用步骤

1. **选择识别目录**
   - 点击"选择目录"按钮
   - 选择包含 `data.json` 和 `personalData` 文件夹的目录

2. **输入API Key**
   - 在API Key输入框中输入您的豆包API密钥
   - 可选：点击"测试连接"验证API是否可用

3. **配置参数**
   - 并发数：同时处理的学号数量（默认1）
   - 重试次数：API失败时的重试次数（默认2）

4. **添加学号**
   - 手动输入学号并点击"添加"
   - 或点击"从文件导入"批量导入学号列表

5. **开始处理**
   - 点击"开始批量识别"按钮
   - 处理过程中可以继续添加学号
   - 可以暂停/恢复或停止处理

6. **查看结果**
   - 右侧日志区域实时显示处理进度
   - 处理完成后可导出结果报告（CSV或JSON格式）

## 打包为exe

### 方式1：使用PyInstaller命令

```bash
# 单文件打包（推荐）
pyinstaller --name "批量预识别工具" --windowed --onefile main.py

# 多文件打包（启动更快）
pyinstaller --name "批量预识别工具" --windowed main.py
```

### 方式2：使用spec文件

```bash
pyinstaller build.spec
```

打包完成后，exe文件位于 `dist/` 目录下。

## 配置文件说明

程序运行后会自动生成 `config.json`，保存您的设置：

- `last_directory`: 上次使用的目录
- `api_key`: API密钥
- `concurrent_workers`: 并发数
- `max_retries`: 重试次数
- `log_level`: 日志级别（0=简要，1=详细）
- `enable_notification`: 是否启用完成通知
- `enable_sound`: 是否启用声音提示

## 目录结构要求

```
your_data_directory/
├── data.json                    # 配置文件
└── personalData/                # 学号文件夹
    ├── 2021001/
    │   ├── 1.jpg
    │   ├── 2.jpg
    │   └── ...
    ├── 2021002/
    │   └── ...
    └── ...
```

### data.json 格式示例

```json
{
  "profiles": {
    "实验1": {
      "expName": "实验名称",
      "prompts": [
        {
          "type": "textRecognition",
          "value": "请识别图片中的数据...",
          "recognitionSource": "1.jpg",
          "model": "doubao-seed-1-6-vision-250815"
        },
        {
          "type": "generateAnswer",
          "value": "请根据以下数据生成答案...",
          "model": "doubao-seed-1-6-flash-250828"
        }
      ]
    }
  }
}
```

## 输出结果

处理完成后，每个学号文件夹下会生成：
- `{学号}_apiRecognizedData.json`

包含所有配置的识别结果和生成的答案。

## 常见问题

### Q: 提示"图片不存在"怎么办？
A: 检查 `personalData/{学号}/` 文件夹下是否有对应的图片文件。

### Q: API调用失败怎么办？
A: 
1. 检查API Key是否正确
2. 检查网络连接
3. 增加重试次数

### Q: 如何提高处理速度？
A: 增加并发数（建议不超过5，避免API限流）

### Q: 处理中断了怎么办？
A: 程序会自动跳过已处理的学号，可以重新添加未完成的学号继续处理。
