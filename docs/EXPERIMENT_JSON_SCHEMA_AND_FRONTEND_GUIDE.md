# 实验数据接入与前端搭建指南 (SOP)

在成功跑通第一个实验（电表的改装）后，我们沉淀了一套核心的**数据驱动UI（Data-Driven UI）**模式。接下来的 9 个实验，需严格遵循本文档的规范进行数据清洗和组件复用，以保证架构的一致性和可维护性。

## 1. 核心设计理念
- **无状态前端**：前端不写死任何具体的实验参数、公式推导逻辑或填空题内容，所有页面结构均由 JSON 配置文件驱动。
- **服务端防线**：前端组件的所有高级交互（一键填空、一键计算、AI识别、生成解答）**仅负责触发状态与鉴权拦截**，必须包含 Loading 过渡态，核心逻辑最终将收束于后端（FastAPI）接口。

## 2. JSON 数据清洗与结构规范
新实验的 JSON 配置必须严格遵守以下拓展结构：

### 2.1 填空与参数映射 (`metaInfo`)
所有变量必须分类注册，以配合前端输入框的状态渲染：
- `fixedIds`: 固定参数（如标准电阻，由后端下发）。
- `computedIds`: 待推导数据。
  - **状态规范**：前端使用 `is-computed` 类名（主题蓝边框与文字），背景保持纯白。
  - **交互规范**：必须**放开只读限制** (`readOnly={false}`)，允许学生在未点击自动计算前手动填入数据。
- `asyncIds`: 后端异步产生的数据。
> **清洗注意**：原 HTML 中的下划线 `___` 必须清洗为独立的输入框对象并赋予唯一 `nodeId`。

### 2.2 图文混排与 AI 视觉 (`inputs` & `ai`)
从旧版实验报告提取的图片分为两类：
- **行内公式图 (`inline: true`)**：前端渲染必须应用 `vertical-align: middle` 和 `margin: 0 4px`，按原生尺寸或缩放后尺寸无缝嵌入段落。
- **块级大图 (`inline: false/undefined`)**：取消原有的高度限制，允许最大高度 `400px`，独立占行居中展示（确保等效电路图清晰可辨）。
- **AI 挂载**：在 `inputs.images` 中定义槽位（如 `IMG_RAW`），并在 `ai.recognition.imageRef` 中绑定，前端据此唤起智能解析组件。

### 2.3 动态数据表格 (`ui.dataTable`)
用于生成底部实验处理区。必须按行 (`rows`)、列 (`cells`) 提供标准数据，前端解析器会自动处理跨行或 LaTeX 公式表头（如带有下标的 $I_x$）。

## 3. 前端组件复用与排版规范
开发新实验详情页时，**严禁重新造轮子**，直接拼装以下基建组件：

1. **`SectionShell` (区块容器)**
   - **排版规范**：不同附加数据板块之间通过自然的 `marginTop: '24px'` 隔开，必要时加顶边线分割，避免滥用厚重的背景 Card 导致嵌套过深。
   - **挂载操作**：涉及计算推导的板块，必须利用其 `extra` 属性（自带 Flex 左右对齐），在小标题最右侧挂载计算数据按钮。
2. **`ExperimentDataTable` (动态表格)**
   - 接收 `ui.dataTable` 数组渲染，自带响应式与输入流控。
3. **`ExperimentImageUploader` (画廊上传区)**
   - **画廊规范**：禁止硬编码限制数量（如 `maxCount`），上传后下方呈现水平滚动的缩略图列表，点击触发主视图缩放，并自带删除功能 (`onRemoveImage`)。

## 4. 按钮交互与高级权限规范 (Plus/Pro)
所有的自动化工具和 AI 辅助功能，必须遵守以下标准：
- **禁用 Small 尺寸**：主功能按钮（尤其是 `GoldButton` 等长文案操作）**严禁使用 `size="small"`**，防止内边距丢失破坏呼吸感。
- **外观标准**：特权按钮使用 `recognize-primary-button` 类名（尊贵蓝），带 `(Plus/Pro)` 或 `(Pro)` 后缀。
- **权限拦截**：
  - 必须经 `capabilities.canUseAssistedFill` 或 `canUseRecognition` 拦截。
  - 阻断提示统一为：`权限拒绝：该功能需要 Plus/Pro (或 Pro) 订阅。`
- **操作反馈**：
  - 点击时必须触发加载（Loading）态。
  - 成功提示带有动态尊享前缀：`` 尊贵的 ${debugRole === 'pro' ? 'Pro' : 'Plus'} 用户， ``
  - 提示文案的后半段必须强化**后端已完成校验**的心智。

## 5. 新实验搭建 CheckList
- [ ] JSON 是否抽离了公式图片并打上 `inline: true`？
- [ ] JSON 的 `nodeId` 是否与原始变量完全匹配？
- [ ] `computedIds` 表单元素是否处于开放编辑（非 readonly）状态且无旧版死板背景色？
- [ ] UI 结构是否实现了完全的自适应，未因滥用 Card 导致多余边白？
