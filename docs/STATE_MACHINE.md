# 实验报告平台状态流转机制 (State Machine)

为了彻底根除不同角色、不同行为导致的数据不一致问题，本系统所有的业务状态流转必须严格遵守以下契约。

## 全局状态字典

### Order (订单)
- `pending_payment`: 待付款（学生点击挂起提交时产生）。
- `paid`: 已支付（管理员确认收款后更新）。
- `rejected`: 已驳回。

### Submission (提交任务)
- `pending_payment`: 待付款挂起中（必须挂载有效的 `order_id`）。
- `pending_recognition`: 待自动识别（已付款或无需付款，等待 AI 识别触发，这是后台列表 [🤖 识别] 按钮的依据）。
- `recognizing`: 正在进行自动识别。
- `reviewing`: 审核中（Reviewer 已认领）。
- `submitting`: 自动填报中（触发了 Playwright）。
- `completed`: 完成。
- `error`: 异常。

## 核心流转规则

### 1. 学生逐单付费流 (按次代劳)
1. 学生在前台调用 `/upload` 上传图片，获得真实的文件路径列表 `image_paths`。
2. 学生调用 `/submit`，并声明 `is_hungup=True`。
   - **后端行为**：生成状态为 `pending_payment` 的 `Order` 和 `Submission`，并且 `Submission` 记录 `image_paths`。
3. 管理员在后台审核该 `Order`，调用 `/verify` 确认收款。
   - **后端行为**：`Order.status` -> `paid`；其关联的 `Submission.status` -> `pending_recognition`（若配置了自动识别则直接进入 `recognizing`）。任务正式流入审核员大厅。

### 2. 学生自主提交流 (Free)
1. 学生在前台界面上传实验相关图片。
2. 学生在页面上完全自主完成固定填空、数据表格填写以及实验主观题解答（**不可使用**一键填空、一键生成式回答等进阶功能）。
3. 学生在页面上点击“临时提交”。
4. **系统行为**：系统会将学生填写好的数据保存一份备份，并将状态流转至自动化处理节点（如 `submitting`）。
5. **自动化行为**：Playwright 根据备份数据自动将内容填写到学校实验报告系统中，并在系统中自动点击“临时提交”按钮。

### 3. 学生自主提交流 (Plus)
1. 学生在仪表盘进入升级套餐，选择购买 Plus 并支付，订单流转到待确认支付。
2. 管理员确认收款，学生账号成功升级为 Plus。
3. 学生进入实验页面，此时**可使用**“一键填空”、“一键识别图标”、“一键生成回答”等功能快速完成报告内容。
4. 学生核对内容后，点击“正式提交”。
5. **系统行为**：系统保存一份最终数据的备份，并进入自动化提交流程。
6. **自动化行为**：Playwright 启动，将确认的数据写回学校实验报告系统，并在系统中自动点击“正式提交”按钮。

### 4. Pro 会员购买与特权流 (全托管)
1. 学生在前台购买 Pro 会员并经由管理员确认支付后，升级为 Pro。
2. 学生在前台仅需上传实验图片后直接发起提交。
3. **系统行为**：任务直接设定为 `pending_recognition` 并进入审核员大厅。后台执行一键填空、识别图片与生成实验问题回答。
4. **人工审核流**：管理员（或 Reviewer）进入任务审核页面，对照原始图片数据，审核系统自动填写的信息和其他一键填写结果是否有问题并进行必要修正。
5. **自动化行为**：管理员审核无误后，点击“临时提交”（或正式提交），触发 Playwright 将修正后的数据写回学校实验报告系统，并自动点击对应按钮提交。

### 5. Admin / Reviewer 跨权代交流
1. Admin 在前台输入他人的学号进行代交。
2. 后端直接跳过所有付款检查，强行将该代交的 `Submission.status` 设定为 `pending_recognition`（或自动识别直接进入 `recognizing`）。
3. Reviewer 能够立刻在任务池中刷新出该任务，并能在界面右侧看到代交时上传的图片。

## 关键拦截防线
- **403 Forbidden 拦截**：任何没有成功支付关联订单、且没有购买 Pro 会员、且非管理员的用户，如果试图伪造请求直接调用 `/submit(is_hungup=False)`，都会被后端强硬拦截，不会产生任何脏数据。
